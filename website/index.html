How it works

From gesture to
subtitle in milliseconds

SignLink watches your caller's video feed, extracts body landmarks, and classifies the sign — all without sending raw video anywhere.

01

Capture

Extension grabs frames from the caller's video element inside Google Meet — no screen recording, no raw video uploaded.

02

Landmark Extraction

MediaPipe Holistic extracts 225 landmark coordinates — both hands + pose — from each frame on the backend.

03

AI Classification

A Bidirectional LSTM reads the landmark sequence and classifies which PSL sign was performed with confidence score.

04

Show Subtitle

The predicted sign appears as a subtitle overlay on YOUR screen in both English and Urdu — the caller sees nothing extra.

Requires backend server

SignLink needs a Python backend running (either locally or on a cloud server) to perform landmark extraction and AI inference. The Download page includes full setup instructions.

🧠 AI Model

Bidirectional LSTM with hidden size 64, 2 layers. Trained on 300-frame normalized landmark sequences using 5-fold cross-validation.

BiLSTM · 256K params

🔄 State Machine

IDLE → SIGNING → PREDICTING pipeline. Motion threshold detects when signing starts and stops. Prevents false positives from body movement.

threshold: 0.015

🎯 TTA Inference

Test-Time Augmentation runs 3 model passes with slight noise and averages probabilities for more stable, reliable predictions.

3× TTA passes

Setup Guide

1

Install Python dependencies

Open terminal in the SignLink folder and run:
pip install -r requirements.txt

2

Start the backend server

Run the FastAPI server locally:
python backend/server.py
Server starts at ws://localhost:8000/ws

3

Install the Chrome extension

Open chrome://extensions → Enable Developer Mode → Click "Load Unpacked" → Select the extension/ folder

4

Open Google Meet

Join any meeting. When your caller signs, SignLink automatically detects it and shows the subtitle on your screen.

5

Supported signs

FAQ

Common questions

Everything you need to know about SignLink.

Does the backend server need to run on my computer?
+
Yes, for local use the Python server must be running on your machine. If you want it to work without this, a cloud-hosted version of the server can be deployed on Railway or Render — the extension just needs to point to that URL instead of localhost.
Is my video sent anywhere?
+
Only compressed frame data is sent to the local backend server via WebSocket. No video is recorded, stored, or sent to any third party. When running locally, all data stays on your machine entirely.
Which video conferencing platforms are supported?
+
Currently optimized for Google Meet. The extension captures from HTML video elements, so it can be adapted to Zoom Web, Microsoft Teams Web, and others with minor code changes.
How accurate is the detection?
+
The model achieves88.7% mean accuracy across 5-fold cross-validation and 100% on the held-out test set. Real-world accuracy depends on lighting, camera angle, and signing speed.
Can more signs be added?
+
Yes. Record videos of the new sign, extract landmarks using the provided scripts, retrain the model with the updated dataset, and replace the model file. The system is designed to scale to more classes.
Does it work without internet?
+
If the backend server is running locally, yes — everything works offline. The extension communicates with localhost only, so no internet connection is required for inference.
